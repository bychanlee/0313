# CLAUDE.md — 031326-YOO (방탈출 서프라이즈 프로젝트)

## 프로젝트 개요

연인에게 주는 서프라이즈 선물용 웹 방탈출 게임.
3개의 퍼즐을 순서대로 풀면 최종적으로 실물 캐리어(여행 선물)를 찾게 되는 구조.

## 컨셉

- **테마**: 고장난 PC / 해킹된 터미널 / 13일의 금요일
- **분위기**: 전체 공포 유지 (분위기 전환 없음, 스토리 없음, 무드와 퍼즐만)
- **텍스트 톤**: 미니멀. 시스템 메시지 스타일 (`ERROR`, `ACCESS DENIED`, `SYSTEM_BREACH` 등)
- **비주얼**: CRT 스캔라인, 글리치, 커서 깜빡임, 비네팅, 랜덤 에러 메시지 부유

## 전체 흐름

```
[PC 브라우저] 퍼즐 1 (로컬 HTML)
    ↓ 완성 → QR코드 표시
    ↓ 폰으로 스캔
[모바일] 퍼즐 2 (GitHub Pages 등 웹 호스팅)
    ↓ 완성
[PC 브라우저] 퍼즐 3 (로컬 HTML 또는 웹 호스팅)
    ↓ 완성 → 캐리어 힌트
[실물] 캐리어 발견
```

## 퍼즐 설계

### 퍼즐 1: `SYSTEM_BREACH` — QR 슬라이드 퍼즐

- **파일**: `puzzle1.html` (로컬 HTML, 브라우저에서 직접 실행)
- **구조**: 21×21 QR코드 그리드. 전체가 화면에 표시됨.
- **메커니즘**:
  - QR코드의 파인더 패턴 3개(좌상, 우상, 좌하 모서리)는 고정
  - 나머지 대부분도 고정 (opacity 낮게 처리)
  - **가운데 3×3 영역만 섞여있는 슬라이드 퍼즐** (8퍼즐 방식, 빈칸 1개)
  - 움직일 수 있는 조각만 초록 펄스 테두리로 강조
- **힌트 시스템**:
  - 올바른 위치의 조각은 밝아짐
  - `RECOVERY: XX%` 완성률 표시
  - (행/열 잠금은 3×3이라 불필요)
- **클리어 조건**: 3×3 완성 → `RECOVERY: 100%` → 동작 멈춤 → QR코드가 스캔 가능한 상태로 확정
- **QR코드**: 2번 퍼즐 URL을 인코딩한 **진짜 스캔 가능한 QR코드**
- **분위기**: `2026-03-13` 상단 표시, 글리치, 스캔라인, 랜덤 에러 메시지
- **답**: 없음 (슬라이드 퍼즐 완성이 곧 클리어)

**⚠️ 핵심 구현 과제**: 현재 QR코드는 가짜 패턴. 실제 스캔 가능한 QR코드를 생성해야 함. `qrcode.js` 등 라이브러리 사용하거나, 2번 퍼즐 URL 확정 후 QR 매트릭스를 하드코딩.

### 퍼즐 2: (답: `0615`)

- **호스팅**: GitHub Pages 등 웹 서버 (QR 스캔으로 접속)
- **기기**: 모바일 (폰에서 플레이)
- **답**: `0615` (6월 15일, 기념일)
- **분위기**: 공포 유지. 고장난 PC / 터미널 톤.
- **퍼즐 타입**: 미정 (논리/암호/패턴 중 택)
- **퍼즐 설계 방향 (이전 논의)**:
  - 에러 로그 속 숫자 단서 추출 + 논리 연산 조합
  - 또는 시스템 시계 고장 → 올바른 시각 조합 → `06:15`
  - 하드코어 난이도. 다중 단계.

### 퍼즐 3: (답: `TRAVEL`)

- **기기**: PC 브라우저 (로컬 HTML 또는 웹 호스팅)
- **답**: `TRAVEL` (영문 6자리)
- **분위기**: 공포 유지.
- **퍼즐 타입**: 미정 (논리/암호/패턴 중 택)
- **퍼즐 설계 방향 (이전 논의)**:
  - 비즈네르 암호 (CIPHER: VIGENERE, KEY: FRIDAY)
  - 숫자열 복호화 → TRAVEL 도출
  - 하드코어 난이도. 깨진 힌트 복원 필요.
